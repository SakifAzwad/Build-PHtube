Tell us the use cases of null and undefined

Null is an assignment value. It can be assigned to a variable as a representation of no value. Undefined means a variable has been declared but has yet not been assigned a value.

The data type of undefined is undefined whereas that of null is object. We can find the datatypes of both undefined and null using the typeof operator.
When used in arithmetic operations, undefined will result in NaN (not a number), whereas null will be converted to 0 behind the screens.
When used in conditional logic both undefined and null will return false.


Use Cases for null:


Explicitly Indicating the Absence of a Value:


null is often used to explicitly represent the absence of a value or the intentional absence of an object's value. It is used when you want to indicate that a variable or property has no meaningful value.


Resetting Object Properties:


You can set an object property to null to indicate that it no longer holds a valid reference or data.


Use Cases for undefined:


Variable Declaration:


Variables that are declared but not assigned a value automatically have the value undefined.


Function Parameters:


When a function is called with fewer arguments than declared parameters, the remaining parameters are undefined.


Accessing Nonexistent Object Properties:


When you try to access an object property that doesn't exist, it returns undefined.


Return Value from Functions:


If a function does not explicitly return a value, it implicitly returns undefined.


Variable Scope:


Variables declared with var are automatically initialized with undefined if not explicitly assigned a value.

What do you mean by REST API?

A REST API (Representational State Transfer Application Programming Interface) is a way to create and interact with web services. It's like a set of rules for building web applications that communicate over the internet. Here are some key points: No Memory: Each time a request is made, it should include all the information needed. The server doesn't remember past requests. Client and Server: The client (e.g., your web browser) and the server (the computer serving the web page) are separate. They talk to each other over the internet. Simple Rules: REST APIs use simple rules for working with data. For example, you can use standard actions like GET (to get data), POST (to send data), PUT (to update data), and DELETE (to delete data). Resources: Everything is treated as a resource with a unique URL (web address). Think of a resource like a piece of data you want to access, like a web page or an image. Different Views: Resources can be shown in different ways. For example, you can view a web page as HTML, JSON, or XML. Easy to Scale: REST makes it easy to expand and grow your web service because of its simplicity and separation between client and server. Layers: You can add extra layers, like caching or security, between the client and server without breaking things. No Memory: The server doesn't remember past requests from the client. Each request must contain all the information needed. Caching: The server can say whether a response can be stored and reused by the client, which can speed things up. Client Control: Clients (like your web browser) can control what they want from the server. Servers don't keep track of client state. In simple terms, RESTful APIs are a set of guidelines for building web services that are easy to use and can communicate with each other over the internet. They're widely used for building websites, mobile apps, and connecting different online services. REST APIs are favored because they are straightforward and work well with the internet's foundation, HTTP.
